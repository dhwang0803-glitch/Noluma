## 작업 내용

0.9.0 릴리즈 후 사용자 환경에서 "중복 태그가 없습니다"만 표시되는 문제를 디버깅하고 수정.

### 문제 1 (0.9.1): SCHEMA_VERSION 미검증
- **증상**: 3-Level 프롬프트로 업그레이드해도 기존 캐시가 유지됨
- **원인**: `isCompatible()`이 `meta.version`을 체크하지 않음
- **수정**: SCHEMA_VERSION 1→2 + isCompatible에 version 비교 추가

### 문제 2 (0.9.2): LLM 실패 시 빈 결과 영구 캐시
- **증상**: 0.9.1 BRAT 업데이트 후 첫 실행에서 LLM API 에러 → 빈 결과 캐시 → 이후 실행 스킵
- **원인**: LLM이 0개 그룹 반환해도 모든 태그를 processedTags에 저장
- **수정**: LLM 결과가 0개일 때 미그룹 singleton을 processedTags에서 제외

## 패턴 분석

### Keep (유지)
- **디버그 로그 삽입 → 즉시 배포 → 사용자 피드백**: 원인을 정확히 좁힘 (newTags:0 → 캐시 미삭제 확인, hasNewSingletons:false → LLM 스킵 확인)
- **로컬 빌드 → vault 복사**: BRAT 릴리즈 사이클 없이 즉시 테스트 가능

### Drop (중단)
- **수동 `gh release create` 후 CI 릴리즈**: CI가 같은 태그로 release를 만들려다 충돌. 앞으로 릴리즈는 태그 push → CI 자동 생성만 사용
- **main 직접 머지**: 핫픽스라 해도 정식 PR 루트를 거쳐야 기록이 남음

### Try (시도)
- **릴리즈 후 smoke test 체크리스트**: 릴리즈 직후 주요 기능 1회 실행 검증 절차 추가
- **캐시 포트에 방어적 설계 패턴화**: version 체크 + 빈결과 방어를 다른 캐시(embeddings, tfidf)에도 적용 검토
